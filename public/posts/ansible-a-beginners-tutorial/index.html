Ansible: A Beginner's Tutorial.

Posted on Feb 14, 2023

First let’s see what chatGPT says about Ansible:

Ansible is an open-source automation tool used for IT tasks such as configuration management, application deployment, and task automation. It allows users to automate repetitive tasks across multiple servers and devices simultaneously, reducing the need for manual intervention and streamlining workflows.

One of the key benefits of Ansible is its agentless architecture, which means that it can manage remote systems without requiring any additional software to be installed on those systems. This makes it easy to deploy and use, and also improves security by minimizing the number of attack surfaces.

Overall, Ansible is a powerful tool for automating IT tasks and improving operational efficiency, making it a popular choice for organizations of all sizes.

Okay, with the definition of what Ansible is, let’s move on to two basic concepts: Playbook and Inventory.

Inventory: An inventory is a file that defines a list of remote hosts or network devices that Ansible can connect to and manage. The inventory file is a simple text file that contains a list of hosts or IP addresses, grouped by different categories or groups. Usually the inventory is in the format of an .ini file.

Playbook: A playbook is a file containing a series of tasks and instructions written in YAML format that are executed on one or more remote hosts. A playbook describes the desired state of a system, which is achieved by executing a series of tasks that perform specific actions, such as installing software, configuring services, or modifying system settings. Playbooks are a key component of Ansible, providing a simple and flexible way to automate complex tasks and manage system configurations. They are designed to be easy to read, write, and maintain.

Okay, with the basic concepts described, let’s practice a little.

For our tests we initially need a virtual machine with an installed system, I chose to use QEMU as my vm and install the Alpine Linux system. But everything we do here can be done with Virtbualbox and Ubuntu, for example. The important thing is to install the system and be able to access it via ssh.

With our VM running we are going to write our inventory, so for this we will create a file called inventory with the following content:

firstserver ansible_host=192.168.122.91 ansible_connection=ssh ansible_user=username ansible_ssh_pass=somepass

[serveralpine] firstserver

Explaining: firstserver = name, like a variable, for your target host. ansible_host = address of your target. ansible_connection = connection type ansible_user = name of the user who will access the machine ansible_ssh_pass = user password [serveralpine] = group name, we can add as many hosts as we want into the group

To test if our inventory is working let’s ping it:

ansible serveralpine -i inventory -m ping
